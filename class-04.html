What is a wireframe?

Wireframing is a practice which is used by UX Designers which allows them to plan and layout the design of their webpage, app or product.

It is a great way for teams of people to be able to collaboratively create a design which will work for every user which may interact with your interface

A commonly used argument of wireframing is that, if a user doesn't know where to go on a plain hand-drawn diagram, then it won't matter how many fancy texts and colours you use on your desired platform.

Typical examples of wireframes

  • Pencil and paper drawn
  • Whiteboard
  • Software developed

Structural process of wireframes

  • Wireframe > Interactive Prototype > Visual > Design
  • Sketch > Code
  • Sketch > Wireframe > Hi-Def Wireframe > Visual > Code
  • Sketch > Wireframe > Visual > Code

Best tools for wireframes

  1. UXPin - it builds responsive, clickable prototypes directly in the browser
  2. InVision - allows feedback directly to the team/users through clickable mock-ups
  3. Wireframe.cc - provides the tech to be able to create quickly in the browser, digital version of pen and paper

6 Steps to make a wireframe

  1. Do your research: Before you even start getting stuff down on pen and paper, you need to have an idea of what you want to do, who will use it and what does it all mean?

  2. Prepare your research for quick reference: Cheatsheets are a great way to get quick information down which you might want to recall at a moments notice later down the line.

  3. Make sure you have your user flow mapped out: You need to know where the user will be coming from, and where you would like them to end up, aka 'User Flows' & 'Information Architecture'

  4. Draft and sketch: Grab a pen/sharpie and ask yourself some simple questions such as,

    - How can you organise the content to support your users goals?

    - Which information is the most important?

    - What should the user first see when arriving on the page?

    - Which buttons or touch points does the user need to click to complete the desired actions?

  5. Add detail and get testing: Add detail however you want and start testing it out. Remember the wireframe is skeleton of the site!

  6. Start turning wireframes into prototypes: Start putting your wireframes onto prototypes such as Sketch and more popular, Figma.

What is HTML?

HTML (HyperText Markup Language) is the code that is used to structure a web page and its content. For example, content could be structured within a set of paragraphs, a list of bulleted points, or using images and data tables.
